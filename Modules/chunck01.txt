Create an AI-Based Garage Recommendation System - a unified mobile app with 4 roles (Driver, Garage, Mechanic, Admin).

PROJECT OVERVIEW:
- Drivers report vehicle issues via AI chatbot → Get garage recommendations OR roadside mechanics
- Garages receive bookings → Manage jobs → Assign mechanics → Track earnings
- Independent Mechanics receive Uber-style job requests → Navigate to customer → Complete repairs
- Admin approves garages/mechanics → Monitors system

TECH STACK:
- Frontend: React Native + TypeScript
- Backend: Node.js + Express.js
- Database: MongoDB with Mongoose
- Real-time: Socket.io (WebSocket)
- AI: Gemini Vision API for diagnostics
- Maps: Google Maps API (or OpenStreetMap/Leaflet as fallback)
- Auth: JWT + Email OTP (nodemailer)

COMPLETE BACKEND SETUP:

1. PROJECT STRUCTURE:
/server
  /models
    - User.js (role: driver|garage|mechanic|admin, email, password, phone, isApproved, isActive)

    - Rating.js (bookingId, raterId, rateeId, rating, review)
    - ChatMessage.js (bookingId, senderId, receiverId, message, timestamp)
  /routes
    - auth.js (register, login, verify-otp, change-password)

2. DATABASE MODELS (Key Fields):
User: email, password, role, phone, otp, otpExpiry, isApproved, isActive
Vehicle: userId, model (Suzuki Alto 2010-2025), year, mileage, transmission, registration
Garage: userId, name, type (Local/Authorized), businessRegNo, address, lat, lng, availabilityStatus (open-available|open-busy|closed), waitTime, mechanics[{name, experience, specialization, isAvailable}], rating
Booking: driverId, garageId/mechanicId, diagnosis {category, type, severity, parts[], repairTime, isDrivable, symptoms[]}, status, billAmount, paymentStatus
Rating: rating (1-5), review

3. AUTH FLOW:
- Register → Send OTP to email → Verify OTP → Set password → Login returns JWT
- Protected routes use authMiddleware
- Role-based access with roleMiddleware
